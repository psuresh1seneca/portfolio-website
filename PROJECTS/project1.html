Project: Deploying a Kubernetes Cluster with Kind on AWS EC2 to Host Containerized Applications

Project Overview

This project involves setting up a containerized application using Kubernetes (K8s) on an Amazon Linux-based EC2 instance. The deployment process leverages Kind (Kubernetes-in-Docker) to create a lightweight K8s cluster, enabling seamless container orchestration. The application consists of two microservices—Cats and Dogs—which will be deployed using Kubernetes manifests, including pods, replica sets, deployments, and services. The applications are exposed using NodePort services to ensure external accessibility. Additionally, Amazon Elastic Container Registry (ECR) is used for secure storage of container images, and AWS Identity and Access Management (IAM) ensures proper access control for the EC2 instance. Terraform is optionally utilized for infrastructure automation.

Technology Purpose
Amazon EC2 Hosts the Kubernetes cluster and containerized applications
Kind Deploys a lightweight Kubernetes cluster on EC2
Kubectl CLI tool to manage Kubernetes cluster and deployments
Amazon ECR Stores and manages container images securely
AWS IAM Grants EC2 access to ECR and other AWS services
Cloud9 IDE Develop and build container images
Terraform Automates infrastructure deployment
