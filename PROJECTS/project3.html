Project: Guestbook Web Application on Amazon EKS

Project Overview

This project focuses on building and deploying a Guestbook Web Application on Amazon EKS (Elastic Kubernetes Service) using Kubernetes components such as Persistent Volumes (PVs), Persistent Volume Claims (PVCs), StorageClasses (SCs), ConfigMaps, and Secrets. The objective is to implement a scalable and resilient architecture that ensures data persistence, secure access management, and dynamic application configuration.
Key Features Implemented:
A PHP-based Guestbook Application that stores user messages in MongoDB.
A frontend service dynamically configured via Kubernetes ConfigMap.
Persistent storage provisioning using Amazon EBS-backed PVs and PVCs.
Deployment of MongoDB as a single pod with a ClusterIP service for internal communication.
LoadBalancer service for public access to the frontend.
Static and dynamic storage provisioning techniques using hostPath and EBS.
Resilience testing by deleting and redeploying MongoDB pods.
Technology Purpose
GitHub Stores application code, Kubernetes manifests, and CI/CD pipeline configurations.
Docker CLI Builds and manages container images for the Guestbook web application and MongoDB.
Kubernetes Manages containerized workloads and automates deployment, scaling, and operations.
Amazon EBS Provides persistent block storage for MongoDB, ensuring data durability.
MongoDB Stores guestbook messages submitted by users.
AWS Cloud9 Provides an IDE for writing code, building images, and deploying applications.
