Project: Configuration Management Automation with Ansible

Project Overview

This project focuses on leveraging Ansible for configuration management automation by deploying a static web server on pre-provisioned Linux-based virtual machines in AWS. The goal is to ensure a repeatable, scalable, and reliable deployment process while utilizing dynamic inventory to adapt to cloud-based ephemeral infrastructure. <
Key Objectives:
Automate Web Server Deployment: Develop an Ansible playbook to install and configure a static website on 2 Ubuntu and 2 Amazon Linux EC2 instances in multiple availability zones.
Ensure Modularity & Scalability: Use Ansible roles, variables, and group-specific configurations to ensure a structured and scalable approach.
Implement Dynamic Inventory: Configure Ansible to use AWS dynamic inventory, allowing seamless adaptation to changes in the infrastructure without modifying static configurations.
Infrastructure as Code (IaC) Approach: Support DevOps best practices by managing infrastructure configurations programmatically.
Security Considerations: Implement preventive and detective security controls to enhance the security posture of the cloud infrastructure.
